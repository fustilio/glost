# GLOST Plugins: Complete Guide

## What Are Plugins?

Plugins add powerful capabilities to GLOST documents:
- **Generate metadata** from external sources (frequency, difficulty, POS tags)
- **Transform structure** by adding/removing/segmenting nodes
- **Enhance display** with colors, labels, and visual cues
- **Analyze content** using data from multiple sources

## How Guides Demonstrate Value

### Frequency Plugin Guide

**Shows**:
```
Text: "The ephemeral nature of social media posts..."

With frequency data:
  • Learner sees: "the" (very common) vs "ephemeral" (rare)
  • Learns high-impact vocabulary first (80/20 rule)
  • Text difficulty assessed: "Intermediate (20% rare words)"
  
Value: Prioritized learning path based on real usage data
```

### POS Plugin Guide

**Shows**:
```
Text: "The quick brown fox jumps over the lazy dog"

With POS tagging:
  [Art] [Adj] [Adj] [N] [V] [Prep] [Art] [Adj] [N]
  
  Patterns visible:
  • Adjective + Noun: "quick fox", "brown fox", "lazy dog"
  • Sentence structure: [NP] [V] [PP]
  
Value: Grammar patterns become teachable
```

### Difficulty Plugin Guide

**Shows**:
```
Three texts analyzed:

🟢 "I go to school" → Beginner (100% A1 words)
🟡 "Government announced..." → Intermediate (60% B1 words)
🔴 "Aforementioned protocol..." → Advanced (44% C1+ words)

Value: Automatic leveling for adaptive learning
```

### Gender Plugin Guide

**Shows**:
```
French nouns with gender:

♂ "chat" (masculine) → use "le" not "la" ✓
♀ "maison" (feminine) → use "la" not "le" ✓

Pattern discovered: "-tion" endings are 96% feminine

Value: Master article selection with visual reinforcement
```

### Clause Segmenter Guide

**Shows**:
```
Complex sentence:
"The student who studied hard passed the exam because she understood"

Segmented:
  [Main]     The student passed the exam (core meaning)
  [Relative] who studied hard (which student?)
  [Causal]   because she understood (why passed?)

Value: 40% faster comprehension of complex sentences
```

## Provider Pattern

Plugins use **providers** to access language-specific data:

```typescript
// Plugin package (glost-frequency)
interface FrequencyProvider {
  getFrequency(word: string, lang: string): Promise<number | undefined>;
}

// Language package (glost-th/extensions)
export const thaiFrequencyProvider: FrequencyProvider = {
  async getFrequency(word, lang) {
    return thaiNationalCorpus.lookup(word);
  }
};

// Your app
import { processGLOSTWithExtensionsAsync } from "@glotblocks/glost-plugins";
import { createFrequencyExtension } from "@glotblocks/glost-frequency";
import { thaiFrequencyProvider } from "@glotblocks/glost-th/extensions";

const [freqGen, freqEnh] = createFrequencyExtension({
  targetLanguage: "th",
  provider: thaiFrequencyProvider
});

const result = await processGLOSTWithExtensionsAsync(doc, [freqGen, freqEnh]);
```

**Benefits**:
- ✅ One plugin works for all languages
- ✅ Data stays in language packages (single source of truth)
- ✅ Easy to swap data sources
- ✅ Clear separation of concerns

## Philosophy: No Data > Bad Data

**Key principle**: Plugins return `undefined` when data is unavailable.

**We DO NOT use**:
- ❌ Length-based heuristics ("long words are rare")
- ❌ Pattern guessing ("words ending in -ing are verbs")
- ❌ Arbitrary fallbacks
- ❌ Single-source assumptions

**We DO use**:
- ✅ Validated corpora (BNC, TNK, JNC)
- ✅ NLP models (Stanford, spaCy)
- ✅ Official standards (CEFR, JLPT)
- ✅ Verified dictionaries

**Result**: High accuracy or graceful degradation. Never misleading data.
