# GLOST Ecosystem

## Overview

GLOST is a modular ecosystem of packages for processing multilingual text with language learning annotations. The architecture follows a layered design with clear separation of concerns.

## Package Architecture

```
LAYER 7: MAIN EXPORT
└── glost                        Main entry point combining all functionality

LAYER 6: CLI & TOOLS
└── glost-cli                    Command-line interface for plugin management

LAYER 5: FRONTEND/INTERFACE
├── glost-react                  React components for rendering GLOST documents
└── glost-dialogue               Dialogue syntax and helper functions

LAYER 4: PROCESSING INFRASTRUCTURE
├── glost-processor              Unified-style processor API
├── glost-registry               Plugin discovery and validation
└── glost-presets                Pre-configured plugin combinations

LAYER 3: LANGUAGE PACKAGES
├── glost-en                     English support (segmentation)
├── glost-ja                     Japanese support (furigana, romaji, etc.)
├── glost-ko                     Korean support (romanization)
└── glost-th                     Thai support (RTGS, segmentation)

LAYER 2: PLUGIN PACKAGES
├── glost-transcription          Pronunciation/transcription data
├── glost-translation            Multilingual glosses
├── glost-frequency              Word frequency metrics
├── glost-difficulty             CEFR/difficulty levels
├── glost-pos                    Part-of-speech tagging
├── glost-gender                 Grammatical gender
├── glost-clause-segmenter       Sentence segmentation
└── glost-inkle                  Interactive fiction integration

LAYER 1: FOUNDATION
├── glost-plugins                Core plugin system architecture
├── glost-core                   Types, nodes, validators, tree utilities
├── glost-common                 Language utilities and shared code
└── glost-utils                  Text parsing and manipulation
```

## How Packages Work Together

### 1. Foundation → Plugins

The foundation layer provides core types and utilities. The plugin system (`glost-plugins`) builds on this to enable extensible processing.

```typescript
// Foundation provides types
import { GLOSTWord, GLOSTSentence } from 'glost-core';

// Plugin system provides extension architecture
import { processGLOST, type GLOSTExtension } from 'glost-plugins';
```

### 2. Plugins → Language Packages

Individual plugins (frequency, transcription, etc.) are composed into language-specific packages that provide a complete solution for each language.

```typescript
// Language package composes multiple plugins
import { thaiFrequencyProvider } from 'glost-th/extensions';
import { createFrequencyExtension } from 'glost-frequency';

const [freqGen, freqEnh] = createFrequencyExtension({
  targetLanguage: 'th',
  provider: thaiFrequencyProvider
});
```

### 3. Processing Infrastructure

The processor, registry, and presets provide the runtime infrastructure for using plugins.

```typescript
import { glost } from 'glost';
import { languageLearningPreset } from 'glost/presets';

const processor = glost()
  .use(languageLearningPreset)
  .freeze();

const result = await processor.process(document);
```

### 4. Interface Packages

Frontend packages consume processed documents for rendering.

```typescript
// React rendering
import { GloSTWord, GloSTSentence } from 'glost-react';

// Dialogue structures
import { createDialogueFromTurns } from 'glost-dialogue';
```

## Community Packages

### Naming Conventions

| Type | Pattern | Example |
|------|---------|---------|
| Language | `glost-{lang}` | `glost-th`, `glost-ja` |
| Data Source | `glost-{lang}-datasource-{source}` | `glost-th-datasource-lexitron` |
| Transcription | `glost-{lang}-transcription-{system}` | `glost-th-transcription-paiboon` |
| Lookup | `glost-{lang}-lookup-{type}-{system}-{source}` | `glost-th-lookup-transcription-paiboon-lexitron` |

### Creating Community Packages

1. Follow [naming conventions](/docs/conventions/naming)
2. Implement provider interfaces from plugin packages
3. Export typed providers
4. Document usage examples
5. Submit PR to add to ecosystem list
